# BI-кейс: обучение контакт-центра по речевой аналитике

В этой папке лежит локальный open-source BI-стенд на базе `Metabase` и `PostgreSQL`. Его можно использовать как учебный конструктор дашбордов: поднять локально, открыть готовый дашборд и показать, как собираются карточки и виджеты.

## Какой BI здесь используется

- `Metabase` - open-source BI-система с конструктором вопросов, графиков и дашбордов.
- `PostgreSQL` - локальная база данных с учебным датасетом.
- `Docker Compose` - способ развернуть оба сервиса одной командой.

По логике показа это хороший аналог демонстрации из класса Tableau / Qlik-подобных инструментов: есть и готовый dashboard, и режим ручной сборки виджета.

## Данные

- Файл: `data/contact_center_speech_analytics.csv`
- Гранулярность: одна строка на сотрудника за одну неделю
- Покрытие: 48 сотрудников и 24 недели наблюдений

## Что измеряем

- выручку от продаж;
- ошибки скрипта, найденные ИИ;
- ошибки, отклоненные супервайзером;
- рекомендации, отправленные в обучение;
- выполнение рекомендаций сотрудниками;
- индекс следования рекомендациям;
- QA score.

## Что нужно для запуска

- macOS, Windows или Linux;
- установленный Docker Desktop или Docker Engine с `docker compose`;
- свободный порт `3000` для Metabase.

## Быстрый запуск

1. Запустите Docker Desktop.
2. Перейдите в эту папку.
3. Выполните `./run_metabase_case.sh`.
4. Дождитесь, пока скрипт:
   - поднимет `postgres` и `metabase`;
   - подключит базу к Metabase;
   - создаст учебные карточки и дашборд;
   - выведет URL, логин и пароль;
   - откроет страницу администратора Metabase в браузере.

Чтобы остановить стенд после показа, выполните `docker compose down`.

## Как показывать сборку виджетов

1. Войдите в Metabase под учеткой из вывода скрипта.
2. Откройте дашборд `Дашборд обучения по речевой аналитике`.
3. Покажите уже собранные карточки:
   - `Выручка по неделям`;
   - `Ошибки ИИ и отклонения супервайзера`;
   - `Командный срез для коучинга`;
   - `Следование рекомендациям и продажи`.
4. Затем покажите живую сборку нового виджета:
   - `+ Создать -> Вопрос`;
   - выбрать базу `Речевая аналитика контакт-центра`;
   - открыть `Notebook editor` для no-code сценария или `Native query` для SQL;
   - собрать агрегацию по неделям, командам или сотрудникам;
   - выбрать тип визуализации `Line`, `Table` или `Scatter`;
   - сохранить и добавить карточку на дашборд через `Add to dashboard`.

## Что удобно проговорить на занятии

- ИИ сначала находит больше ошибок, но супервайзер отклоняет заметную долю.
- Со временем модель и правила обучения стабилизируются, поэтому отклонений становится меньше.
- На уровне сотрудников видно связь: лучшее следование рекомендациям сочетается с ростом продаж.
- BI удобен, когда нужен быстрый дашборд без написания кода и с возможностью собирать виджеты прямо на встрече.
